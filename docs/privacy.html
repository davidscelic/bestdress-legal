Privacy Policy

Effective June 29, 2026

App: BestDress (iOS) · Bundle ID: com.bestdress.ios

Summary

BestDress is a local-first wardrobe app. In the default App Store build (no cloud AI API key configured at compile time), your wardrobe photos, profile details, ratings, and occasions are stored on your device only. We do not operate a BestDress user account system, backend database, advertising network, analytics SDK, or crash-reporting SDK in the current codebase.

If a build is compiled with a cloud vision API key (BESTDRESS_AI_KEY), photos you choose to analyze may be transmitted to that configured provider for AI processing, as described below.

1. Information we process

1.1 Information you provide

DataExamplesDefault build
ProfileFirst name, style direction, preferences, goalsOn device
Sizing (optional)Height, weight, waist, chest, inseam, clothing sizesOn device
WardrobeItem details and photosOn device
Outfit ratingsPhotos, occasion, dress code, notes, resultsOn device
OccasionsName, dress code, theme, location (typed), notesOn device
AvatarOptional profile photoOn device

We do not collect email addresses, passwords, or phone numbers. There is no in-app login to a BestDress server.

1.2 Information generated on your device

  • Style memory — local counts and defaults to improve suggestions; never uploaded.
  • AI scan usage counters — local quota enforcement when cloud AI is enabled; never uploaded.
  • Subscription state — trial date and tier stored locally.

1.3 Subscriptions

Release builds use Apple StoreKit for in-app purchases. Apple processes payment. BestDress stores subscription tier and trial status on your device. Manage subscriptions at Apple Subscriptions.

1.4 Photos and camera

Camera and photo library access is requested only when you add or rate an outfit. Images are stored in the app’s private documents directory and resized on import (max width 1600 px).

2. AI photo processing

2.1 Default build (no API key)

Without BESTDRESS_AI_KEY at build time:

  • Outfit rating uses an on-device mock engine that does not analyze pixels.
  • Closet inference uses an on-device mock based on filename, profile, and closet context.
  • No photos are sent to BestDress or third-party AI servers.

2.2 Optional cloud vision builds

With BESTDRESS_AI_KEY configured at build time, explicit user actions may send images (base64 JPEG) and text to a configurable endpoint (default: OpenAI-compatible chat completions). Transmission is not background. On failure or quota limits, the app falls back to on-device mocks. The configured provider’s privacy policy applies to that processing.

3. What we do not use

The current codebase does not integrate Firebase, Crashlytics, AdMob, RevenueCat, packaged OpenAI/Gemini/Claude SDKs, Supabase, AWS SDKs, push notifications, third-party auth, or cross-app tracking.

4. Storage and retention

Data remains on your device until you use Erase (Me tab) or uninstall the app. We do not operate cloud backup or sync.

5. Data sharing

Default build: No third-party sharing except Apple for purchases.

API-key builds: Images and text sent for AI analysis go to the configured vision API provider. We do not sell personal information.

6. Your rights (GDPR / CCPA)

Use Erase or uninstall to delete local data. California residents may contact us with privacy requests. GDPR users: data is processed to provide app functionality and based on permissions and choices you make.

7. Children’s privacy

BestDress is not directed at children under 13. We do not knowingly collect children’s information. The app does not implement an age gate.

8. Security

Default data is protected by your device OS. API-key builds use HTTPS to the configured endpoint.

9. International transfers

Default builds keep data on device. API-key builds may send data to provider servers outside your country.

10. Changes

We may update this policy. The effective date above will change when we do.

11. Contact

See https://bestdress.app for contact information, or the BestDress project repository for developer distributions.
